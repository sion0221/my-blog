## ğŸ“œ Coding Principles (Strictly Follow)

1. **Design System (Modern Dark)**:
   - Base Background: `#0f172a` (Slate-900/950)
   - Accent: Blue/Emerald Gradients
   - Concept: Glassmorphism, Sleek Typography (Inter), Dark & Modern.
   - Use `class` for `.astro` files and `className` for `.tsx` files.
2. **Design Preservation**: Do NOT modify existing Tailwind CSS classes or UI design unless explicitly requested by the user.
3. **Minimal Refactoring**: Do NOT refactor existing code arbitrarily. Focus only on implementing the requested features.
4. **Naming Convention**:
   - **Components**: PascalCase (e.g., `PostCard.tsx`)
   - **Files/Folders**: kebab-case (e.g., `post-list.ts`)
   - **Booleans**: Prefix with `is-` or `has-` (e.g., `isLoaded`, `hasToken`)
5. **Module System**: All exports must use **Named Export** as a rule. (`export const ...`)
6. **Path Aliases**: Use `@/` for all internal imports (e.g., `@/lib/supabase`). Do NOT use relative paths like `../../`.
7. **No Blind Creation**: Always confirm with the user before creating new files or folders to ensure consistency with the current structure.
8. **No Any Policy**: Never use the `any` type. Always define specific interfaces or use `unknown` with type guards/assertion.
